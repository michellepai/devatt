1. Introduction

The Speech API provides two methods for transcribing audio into text and one method for rendering text into audio.

  • The Speech To Text method transcribes audio data into text based on contexts specified by the app request, using a global dictionary for grammar that is hosted in the Speech engine.

  • The Speech To Text Custom method transcribes audio data into text based upon developer-specified grammar or hints, using industry standard speech synthesis formats.

  • The Text To Speech method converts text to speech audio formats.
Considerations

The RESTful Speech API provided by AT&T has the following considerations for you to keep in mind.

The Speech To Text method.

  • Accepts only single-channel audio formats.
  • Uses the X-SpeechContext header parameter to provide context for more accurate results.
  • Limited to one minute of audio per request. If the VoiceMail context is used, then the limit is four minutes of audio per request.
  • Accepts audio data file or streaming (chunked) audio data as input.
  • Most contexts support English only. The Generic context supports English or Spanish.
  • If the audio data is not understood, then in most cases the HTTP status code of 200 OK is returned and the Status parameter is set to Speech Not Recognized.
  • Failed requests are not retried by the API Gateway. Therefore, you must specify that your app handles this.

The Speech To Text Custom method.

  • Requires the unique scope value of STTC.
  • Differs from the Speech To Text method in that you may supply your own grammar or hints for more accurate results.
  • Uses the X-SpeechContext parameter to differentiate between GammarList and GenericHints style requests.
  • Supports an extended list of 19 languages.

The Text To Speech method.

  • Requires the unique scope value of TTS.
  • Accepts plain text or XML-based Speech Synthesis Markup Language (SSML) as input.
  • Returns binary audio data.
  • Is limited to 500 bytes of text for processing.
  • Supports English or Spanish with male or female synthesized voices.

In all cases, Failed requests are not retried by the API Gateway. Therefore, you must specify that your app handles this.

Provisioning

The RESTful Speech API provided by AT&T requires that you provision your app as follows.

For the Speech To Text method, you must register the SPEECH API scope for your app account on the AT&T Developer Program web site in order to utilize this API.

For the Speech To Text Custom method, you must register the STTC API scope for your app account on the AT&T Developer Program web site in order to utilize this API.

For the Text To Speech method, you must register the TTS API scope for your app account on the AT&T Developer Program web site in order to utilize this API.

2. OAuth

The APIs provided by AT&T use the OAuth 2.0 framework, and an OAuth access token must be included in each request to the API Gateway. To obtain an OAuth access token, use the App Key and App Secret from the My Apps section after registering and signing into your API Platform account.

You may authorize access for this API by using the following scopes.

  • "SPEECH" for the Speech To Text method.
  • "STTC" for the Speech To Text Custom method.
  • "TTS" for the Text To Speech method.

To obtain your OAuth access token run the following command from the Command Line Interface window and replace "{{APP_KEY}}", "{{APP_SECRET}}", and "{{API_SCOPES}}" with the App Key, App Secret, and the API scopes you wish to access as follows.

Get Access Token method using the cURL command:

copy 
curl https://api.att.com/oauth/token
    --request POST
    --insecure
    --data "client_id={{APP_KEY}}&client_secret={{APP_SECRET}}&grant_type=client_credentials&scope={{API_SCOPES}}"

A successful response from the API Gateway contains an OAuth access token, the expiration period, and a refresh token as displayed in the following example.

Response from the API Gateway for the Get Access Token method:

{
    "access_token":"{{OAUTH_ACCESS_TOKEN}}",
    "token_type": "bearer",
    "expires_in":"{{EXPIRATION_PERIOD}}",
    "refresh_token":"{{REFRESH_TOKEN}}"
}

Extract the value from "{{OAUTH_ACCESS_TOKEN}}" to use in the Authorization parameter for requests to the API Gateway. When sending an request to the API Gateway a successfully constructed Authorization parameter looks like the following.

Authorization parameter for the HTTP header:

Authorization: Bearer {{OAUTH_ACCESS_TOKEN}}

For more information about refresh_token and other authentication options, OAuth 2.0 API.

3. Resources
Speech To Text
Overview

The Speech To Text method transcribes audio data files into text-based output. The Speech API applies an API-specific lexicon and transcribes the original text into an enhanced text output. The original and the enhanced text outputs are both returned as separate parameter values in the response. The supported audio formats are:

  • 16-bit PCM WAV, linear coding, single channel, 8 kHz sampling
  • 16-bit PCM WAV, ulaw coding, single channel, 8 kHz sampling
  • 16-bit PCM WAV, linear coding, single channel, 16 kHz sampling
  • 16-bit PCM WAV, ulaw coding, single channel, 16 kHz sampling
  • AMR (narrowband), 12.2 kbit/s, 8 kHz sampling
  • AMR-WB (wideband), 12.65 kbit/s, 16khz sampling
  • OGG, speex encoding, 8kHz sampling
  • OGG, speex encoding, 16kHz sampling
  • Raw, linear coding, little-endian byte order, 8kHz sampling
  • Raw, linear coding, big-endian byte order, 8kHz sampling
  • Raw, ulaw coding, little-endian byte order, 8kHz sampling
  • Raw, ulaw coding, big-endian byte order, 8kHz sampling
  • Raw, linear coding, little-endian byte order, 16kHz sampling
  • Raw, linear coding, big-endian byte order, 16kHz sampling
  • Raw, ulaw coding, little-endian byte order, 16kHz sampling
  • Raw, ulaw coding, big-endian byte order, 16kHz sampling

Specifies the focus user interface (UI) screen or widget of the app for this request. This parameter is an X-Arg parameter.
ClientSdk String Header
Specifies the client SDK. The Speech SDK for native apps provided by AT&T adds this X-Arg parameter automatically, and provides the client app the ability to opt-out of including the parameter. Other SDKs, such as Sencha or Microsoft, should do the same. The syntax for this parameter is as follows.
  • {library}-{platform}-{version}
Each segment matches the regular expression as follows.
  • [A-Za-z1-9_.]+
The segments have the following interpretations:
  • library: The name of the library. This value is determined by the SDK developer. The Speech SDK for native apps provided by AT&T uses the value "ATTSpeechKit". Other library names may include SenchaSpeechSDK, MicrosoftSpeechSDK, and so on.
  • platform: The platform running the library. For SDKs running on the device, this would be Android, Android_Emulator, and non-Android OSs. Future native platforms might include BlackBerry and Windows_RT. For server-side SDKs like Sencha and Microsoft, the platform should be the word Server followed by an underscore and the name of the server platform, such as Server_PHP, Server_Rails.
  • version: The version number of the library. This value is determined by the SDK developer. The Speech SDK for native apps provided by AT&T uses the format of {major}.{minor}.{release}.{build}.
This parameter is an X-Arg parameter.
DeviceType String Header
Specifies the device name, preferably from a standard API on the device itself, such as SGHT999 or SAMSUNG-SGH-I897. This parameter is an X-Arg parameter.
DeviceOs String Header
Specifies the version of the operating system (OS) for the device type. This parameter value comes from a standard API on the device or is auto-filled by an SDK. The format for this parameter is as follows.
  • {name}-{version}
Each segment matches the regular expression as follows.
  • [A-Za-z1-9_.]+
The segments have the following interpretations:
  • name: Specifies the name of the OS, with spaces and other non-alphanumeric characters in the name converted to underscores, such as Android or Windows_RT.
  • version: Specifies the version of the OS, as provided by the OS. The format for this is {{major}}.{{minor}}.{{release}}.{{build}}.
This parameter is an X-Arg parameter.
DeviceTime String Header
Specifies the local device time of the request. The format for this parameter is YYYY-MM-DD followed by a single space ( ) followed by HH:MM:SS followed by a single space ( ) followed by TZ. A sample value may be displayed as follows: 2012-08-08 07:08:10 EDT. This parameter is an X-Arg parameter.

Speech To Text Custom
Overview

The Speech To Text Custom method transcribes audio data files into text-based output using developer customized grammar, hints, or both. The request is always a MIME multi-part message, using Speech Recognition Grammar Specification (SRGS) impementations which may use Context Free Grammar alone or in conjunction with the Generic speech engine. The following MIME part types are supported

  • x-dictionary: The optional PLS data of the following type.
    • application/pls+xml
  • x-grammar: The required SRGS part of the following type.
    • application/srgs+xml
  • x-voice: The binary audio content data to be transcribed.
The supported audio formats are:
  • 16-bit PCM WAV, linear coding, single channel, 8 kHz sampling
  • 16-bit PCM WAV, ulaw coding, single channel, 8 kHz sampling
  • 16-bit PCM WAV, linear coding, single channel, 16 kHz sampling
  • 16-bit PCM WAV, ulaw coding, single channel, 16 kHz sampling
  • AMR (narrowband), 12.2 kbit/s, 8 kHz sampling
  • AMR-WB (wideband), 12.65 kbit/s, 16khz sampling
  • OGG, speex encoding, 8kHz sampling
  • OGG, speex encoding, 16kHz sampling
  • Raw, linear coding, little-endian byte order, 8kHz sampling
  • Raw, linear coding, big-endian byte order, 8kHz sampling
  • Raw, ulaw coding, little-endian byte order, 8kHz sampling
  • Raw, ulaw coding, big-endian byte order, 8kHz sampling
  • Raw, linear coding, little-endian byte order, 16kHz sampling
  • Raw, linear coding, big-endian byte order, 16kHz sampling
  • Raw, ulaw coding, little-endian byte order, 16kHz sampling
  • Raw, ulaw coding, big-endian byte order, 16kHz sampling

        string oAuthEndPoint = "https://api.att.com/oauth/token";
        string oAuthHTTPVerb = "POST";
        string oAuthAccept = "application/json";
        string oAuthContentType = "application/x-www-form-urlencoded";
        string oAuthclient_id = ""; //Update with registered Speech Service Application Id/Key.
        string oAuthclient_secret = ""; //Update with registered Speech Service Secret Key/Id.
        string oAuthScope = "SPEECH";
        string oAuthBody = "client_id=" + HttpUtility.UrlEncode(oAuthclient_id).ToString() +
                                                "&client_secret=" + HttpUtility.UrlEncode(oAuthclient_secret).ToString() +
                                                "&grant_type=" + HttpUtility.UrlEncode("client_credentials").ToString() +
                                                "&scope=" + HttpUtility.UrlEncode(oAuthScope).ToString();
        // Send web request and handle web response 
        HttpWebRequest webRequest = null;
        HttpWebResponse webResponse = null;
        GetAccessTokenResponse getAccessTokenResponse = null;
        try
        {
            webRequest = (HttpWebRequest)System.Net.WebRequest.Create(oAuthEndPoint);
            if (!string.IsNullOrEmpty(proxyIp))
            {
                webRequest.Proxy = new WebProxy(proxyIp, proxyPort);
            }
            webRequest.ContentType = oAuthContentType;
            webRequest.Accept = oAuthAccept;
            webRequest.Method = oAuthHTTPVerb;
            byte[] postBytes = null;
            if (!string.IsNullOrEmpty(oAuthBody))
            {
                UTF8Encoding encoding = new UTF8Encoding();

                postBytes = encoding.GetBytes(oAuthBody);
            }
            if (postBytes != null)
            {
                webRequest.KeepAlive = true;
                webRequest.ContentLength = postBytes.Length;
                Stream postStream = webRequest.GetRequestStream();
                postStream.Write(postBytes, 0, postBytes.Length);
                postStream.Close();
            }

            webResponse = (HttpWebResponse)webRequest.GetResponse();
            using (StreamReader sr2 = new StreamReader(webResponse.GetResponseStream()))
            {
                apiResponseString = sr2.ReadToEnd();
                getAccessTokenResponse = new GetAccessTokenResponse();
                JavaScriptSerializer deserializeJsonObject = new JavaScriptSerializer();
                getAccessTokenResponse = (GetAccessTokenResponse)deserializeJsonObject.Deserialize(apiResponseString, typeof(GetAccessTokenResponse));
                sr2.Close();
            }
            webRequest = null;
            webResponse.Close();
            // Get Access Token End 

            // Send Speech Start
            string speechServiceEndPoint = "https://api.att.com/speech/v3/speechToText";
            string speechFileToDecode = Request.MapPath("~\\audio\\california.amr"); //Update with the path of the speech file
            string speechServiceContentType = "audio/amr"; //Update based on the type of the file, update content type.
            string xSpeechContext = "Generic";
            string xArgs = "ClientApp=NoteTaker,ClientVersion=1.0.1,DeviceType=Android";
            string speechServiceHTTPVerb = "POST";
            string speechServiceAccept = "application/json";
            string speechServiceAuthorization = getAccessTokenResponse.access_token;
            NameValueCollection headers = new NameValueCollection();
            headers.Add("Authorization", "Bearer " + speechServiceAuthorization);
            headers.Add("X-SpeechContext", xSpeechContext);
            headers.Add("X-Arg", xArgs);
            FileStream audioFileStream = new FileStream(speechFileToDecode, FileMode.Open, FileAccess.Read);
            BinaryReader reader = new BinaryReader(audioFileStream);
            byte[] binaryData = reader.ReadBytes((int)audioFileStream.Length);
            reader.Close();
            audioFileStream.Close();
            webRequest = (HttpWebRequest)System.Net.WebRequest.Create(speechServiceEndPoint);
            if (!string.IsNullOrEmpty(proxyIp))
            {
                webRequest.Proxy = new WebProxy(proxyIp, proxyPort);
            }
            webRequest.ContentType = speechServiceContentType;
            webRequest.Accept = speechServiceAccept;
            webRequest.Method = speechServiceHTTPVerb;
            webRequest.Headers.Add(headers);
            webRequest.ContentLength = binaryData.Length;
            webRequest.KeepAlive = true;
            Stream speechStream = webRequest.GetRequestStream();
            speechStream.Write(binaryData, 0, binaryData.Length);
            speechStream.Close();

            webResponse = (HttpWebResponse)webRequest.GetResponse();
            using (StreamReader sr2 = new StreamReader(webResponse.GetResponseStream()))
            {
                apiResponseString = sr2.ReadToEnd();
                JavaScriptSerializer deserializeJsonObject = new JavaScriptSerializer();
                SpeechResponse deserializedJsonObj = (SpeechResponse)deserializeJsonObject.Deserialize(apiResponseString, typeof(SpeechResponse));
                sr2.Close();
            }
            // Send Speech End
        }
        catch (WebException ex)
        {
            try
            {
                if (null != ex.Response)
                {
                    using (StreamReader sr2 = new StreamReader(ex.Response.GetResponseStream()))
                    {
                        apiErrorResponse = sr2.ReadToEnd();
                        sr2.Close();
                    }
                }
            }
            catch
            {
                apiErrorResponse = "Unable to get response";
            }
        }
        catch (Exception ex)
        {
            apiErrorResponse = ex.ToString();
        }
        finally
        {
            webRequest = null;
            if (webResponse != null)
                webResponse.Close();
            webResponse = null;
        }

    }
